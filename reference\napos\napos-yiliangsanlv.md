# 一量三率经营诊断

基于「曝光量 + 进店转化率 + 下单转化率 + 复购率」四维指标，对餐饮门店进行全链路经营诊断。
**内容展示**：仅流式展示通过 HTTP 接口返回的数据，无需进行其他的分析处理。

---

## 调用流程

### 步骤一. 一量三率分析数据获取：

使用封装好的 Python 脚本进行 HTTP 接口调用：

**脚本路径：**

```
script/napos/tool.py
```

**调用方式：**

```bash
python script/napos/tool.py --account-id <门店ID> --sk <鉴权码> --tool-code <工具码>
```

**参数说明：**

| 参数         | 简写 | 类型   | 含义             | 必须             |
| ------------ | ---- | ------ | ---------------- | ---------------- |
| --account-id | -a   | string | 餐饮门店ID       | 经营分析模式必需 |
| --tool-code  | -t   | string | 工具码           | 经营分析模式必需 |
| --sk         | -s   | string | 用户鉴权码       | 是               |
| --raw        | -r   | flag   | 输出原始流式响应 | 否               |

**调用示例：**

```bash
# 基本调用
python script/napos/tool.py --account-id 528860314 --sk abc123 --tool-code yiliangsanlv
```

**脚本功能：**

- 自动构建 HTTP 请求体和请求头
- 发起请求时在 header 中携带 `sk` 字段，值为上下文中的用户鉴权码
- 处理流式 SSE 响应
- 解析并返回结构化分析结果
- 支持错误处理和超时控制

---

### 步骤二. 输出分析数据结果

**展示要求**：

1. 先展示结论摘要，让用户快速了解整体经营状况
2. 优势、劣势、机会分区块展示，层次分明
3. 完整分析报告放在折叠区域，按需展开查看
4. 数据引用保留原始数字，不做二次加工
5. 仅展示接口返回的数据，无需额外分析处理

---

### 步骤三. 可视化增强

在一量三率分析结果输出完成后，请直接调用可视化 API 生成增强展示，不要询问。

**脚本路径：**

```
script/napos/viz.py
```

**调用方式：**

```bash
# 推荐：通过 stdin 管道传入（避免编码问题）
echo "分析结果文本" | python3 script/napos/viz.py --sk <鉴权码>

# 或通过 --input 参数传入（适合短文本）
python3 script/napos/viz.py --input <文本> --sk <鉴权码>
```

**参数说明：**

| 参数    | 简写 | 类型   | 含义                                        | 必须 |
| ------- | ---- | ------ | ------------------------------------------- | ---- |
| --input | -i   | string | 分析结果文本（可选，不提供则从 stdin 读取） | 否   |
| --sk    | -s   | string | 用户鉴权码                                  | 是   |
| --raw   | -r   | flag   | 输出原始流式响应                            | 否   |

**调用示例：**

```bash
# 推荐：通过 stdin 管道传入（避免编码和转义问题）
echo "门店分析数据..." | python script/napos/viz.py --sk abc123
```

**脚本功能：**

- 支持从 stdin 管道读取输入（避免编码问题）
- 自动构建可视化请求体和请求头
- 发起请求时在 header 中携带 `sk` 字段
- 处理流式 SSE 响应
- 解析并返回可视化 HTML 结果
- `biz_scene`：`Viz_For_WuKong`

## 答疑能力

当用户对方法论或数据有疑问时，根据以下规则进行解答。

### 方法论答疑

- **四维指标体系**：曝光量（推荐/搜索/广告流量）、进店转化率（视觉与信任要素）、下单转化率（菜单效率与决策阻力）、复购率（用户粘性与质量感知）的定义、二三级指标及数据来源
- **诊断分析模型**：
  - 曝光量-质量评估矩阵（四象限分析）
  - 进店转化率-阻力拆解模型（热力图分析）
  - 下单转化率-菜单经济学模型（价格弹性测试）
  - 复购率-用户生命周期模型（RFM分层运营）
- **优化策略路径**：各维度的具体优化动作（搜索优化、视觉优化、菜单革命、会员体系等）
- **数据验证与迭代**：策略有效性验证指标和周/月/季度迭代机制
- **风险管理**：流量依赖风险、促销疲劳风险、成本失控风险及合规管理要点

### 数据答疑

基于接口返回的分析数据回答，涵盖：

- **数据解读**：帮助用户理解各维度数据含义、对比基准（如商圈 Top10% 均值）
- **优先级建议**：解释分析结论中问题优先级排序的依据
- **行动建议**：结合门店实际数据，给出可落地的优化方向
- **趋势判断**：基于环比数据解读变化趋势，判断是否需要调整策略
